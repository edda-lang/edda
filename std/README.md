# std

The Edda standard library — 113 packages under `lib/<subsystem>/<package>/`, imported by physical path: `lib/io/stream/` is `std.io.stream`, `lib/core/option/` is `std.core.option`.

Subsystems: `archive`, `bytes`, `collections`, `compress`, `core`, `crypto`, `encoding`, `graphics`, `io`, `ipc`, `math`, `mem`, `net`, `os`, `serde`, `task`, `testing`, `text`, `time`.

Effects and capabilities are part of every API: allocating functions take an `Allocator` and raise `err: alloc.AllocError`; filesystem reads take a `ReadOnlyFilesystem`; there are no ambient globals. Error types are path-qualified in effect rows (`err: stream.IoError`, `err: fs.FsError`).

Each package's surface is its `index.toon` structure map, regenerated by `edda build`. The language reference is [`../AGENTS.md`](../AGENTS.md); everyday entry points (streams, filesystem, strings, parsing) are summarized there.
